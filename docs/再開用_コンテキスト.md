**最終更新**: 2026年3月4日

## プロジェクト概要

**管理栄養士業務システム（kyuushoku-system）**  
保育園3〜5歳児向け給食管理。献立作成・栄養計算・発注集計・まとめ買い。

- **リポジトリ**: https://github.com/kuroroyumi-wq/kyuushoku-system
- **構成**: monorepo（phase1_python, backend_go, frontend_web, data, tools, docs）

---

## 技術スタック

| 層 | 技術 |
|----|------|
| バックエンド | Go, SQLite, REST API |
| フロントエンド | React, TypeScript, Vite |
| データ | CSV → SQLite（migrate_csv_to_sqlite.py） |

---

## 実装済み機能

- **献立一覧**: 月別表示
- **献立入力**: 登録・更新（日付変更で既存献立を読み込み）
- **栄養計算**: 日付指定で栄養価表示
- **発注集計**: 1日単位 / 期間、調味料除外オプション
- **まとめ買い**: 米・調味料・乾物の発注目安（bulk_purchase_guide）
- **認証**: ログイン・初回登録（JWT）

---

## 起動手順

```bash
# 1. データ投入（初回のみ）
python3 tools/migrate_csv_to_sqlite.py

# 1b. 既存DBを Phase4 対応する場合
python3 tools/migrate_add_facility.py
python3 tools/migrate_add_users.py

# 2. API サーバ
cd backend_go && ./menu-system serve
# → http://localhost:8080

# 3. フロントエンド（別ターミナル）
cd frontend_web && npm install && npm run dev
# → http://localhost:5173
```

**PostgreSQL 使用時**: `DATABASE_URL=postgres://...` を設定すると PostgreSQL に接続。移行は `python3 tools/migrate_sqlite_to_pg.py`

---

## トラブルシューティング

| 現象 | 対処 |
|------|------|
| ポート8080使用中 | `lsof -i :8080` → `kill <PID>` |
| DB読み取り専用 | `chmod 644 data/menu.db` |
| 日付エラー | 日付は YYYY-MM-DD 形式で正規化済み |
| 初回ログイン | /login で「登録」をクリックし、メール・パスワード（6文字以上）で初回ユーザー作成 |
