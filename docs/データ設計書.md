# 管理栄養士業務システム データ設計書

**版数**：1.0  
**作成日**：2025年3月3日  
**参照**：日本食品標準成分表2020年版（八訂）増補2023年

---

## 1. 設計方針

### 1.1 食材と料理の分離

| 概念 | 説明 | 用途 |
|------|------|------|
| **食材** | 生の原材料（米、豚肉、豆腐など） | 栄養価計算・発注指示書 |
| **料理** | 調理済みメニュー（豚の生姜焼き、味噌汁など） | 献立作成・献立表 |
| **レシピ** | 料理と食材の対応関係（使用量） | 栄養価計算・発注集計 |

食材マスタは**生食材のみ**を登録し、料理の栄養価はレシピから**計算**する。これにより発注指示書で正確な食材数量を集計できる。

### 1.2 データ参照元

- **栄養価**：文部科学省「日本食品標準成分表2020年版（八訂）」および「増補2023年」
- **出典**：https://fooddb.mext.go.jp/

## 3. 食材マスタ（ingredients.csv）

### 3.1 役割

献立に使用する**生食材**の栄養価を管理する。発注指示書の集計元となる。

### 3.2 カラム定義

| カラム名 | 型 | 必須 | 説明 |
|----------|-----|------|------|
| id | 整数 | ○ | 食材ID（一意） |
| name | 文字列 | ○ | 食材名 |
| ingredient_category | 文字列 | ○ | 食材の分類（穀類/肉類/魚介類/卵/乳製品/豆類/野菜/海藻/果実/調味料） |
| energy | 数値 | ○ | エネルギー（100gあたり kcal） |
| protein | 数値 | ○ | たんぱく質（100gあたり g） |
| fat | 数値 | ○ | 脂質（100gあたり g） |
| carbohydrate | 数値 | ○ | 炭水化物（100gあたり g） |
| salt | 数値 | ○ | 食塩相当量（100gあたり g）※減塩指導・施設向け |
| unit | 文字列 | ○ | 単位（gで統一。mlは使用しない） |
| waste_rate | 数値 | ○ | 廃棄率（0〜1、初期値0） |
| note | 文字列 | - | 備考（アレルギー、調理上の注意など） |

### 3.3 食材分類一覧

| 分類コード | 説明 | 例 |
|------------|------|-----|
| 穀類 | 米、パン、麺類 | ご飯、食パン、うどん |
| 肉類 | 畜肉 | 豚もも肉、鶏むね肉 |
| 魚介類 | 魚、貝類 | 鮭、サバ |
| 卵 | 鶏卵など | 鶏卵 |
| 乳製品 | 牛乳、ヨーグルト | プレーンヨーグルト |
| 豆類 | 豆腐、納豆 | 木綿豆腐 |
| 野菜 | 野菜類 | ほうれん草、ごぼう |
| 海藻 | わかめ、のり | 乾燥わかめ |
| 果実 | 果物 | りんご |
| 調味料 | 醤油、味噌、砂糖 | 醤油、味噌 |

## 4. 献立候補（dishes.csv）

### 4.1 役割

献立作成時に選択する**料理**の一覧。主食・主菜・副菜・汁物・デザートの5カテゴリで管理。

### 4.2 カラム定義

| カラム名 | 型 | 必須 | 説明 |
|----------|-----|------|------|
| id | 整数 | ○ | 料理ID（一意） |
| name | 文字列 | ○ | 料理名 |
| menu_category | 文字列 | ○ | 献立カテゴリ（主食/主菜/副菜/汁物/デザート） |
| serving_size | 整数 | ○ | 1人分の総量（g）※目安 |
| note | 文字列 | - | 備考（調理のポイント、アレルギー等） |

### 4.3 栄養価の算出

料理の栄養価は**レシピから自動計算**する。

```
料理の栄養価 = Σ（各食材の栄養価 × 使用量(g) / 100）
```

---

## 5. レシピ（recipe.csv）

### 5.1 役割

料理に使用する**食材と使用量**を定義する。複数食材の組み合わせに対応。

### 5.2 カラム定義

| カラム名 | 型 | 必須 | 説明 |
|----------|-----|------|------|
| dish_id | 整数 | ○ | 料理ID（dishes.id を参照） |
| ingredient_id | 整数 | ○ | 食材ID（ingredients.id を参照） |
| amount | 数値 | ○ | 1人分の使用量（g） |

※1料理に複数行で複数食材を登録する。

### 5.3 使用例

| dish_id | ingredient_id | amount |
|---------|---------------|--------|
| 4 | 4 | 80 |
| 4 | 13 | 5 |
| 4 | 14 | 10 |

→ 料理ID4（豚の生姜焼き）は、豚肉80g、しょうが5g、醤油10g を使用

---

## 6. データの流れ

```
[ingredients.csv]    生食材の栄養価（100gあたり）
        ↓
[recipe.csv]         料理ごとの食材使用量
        ↓
[dishes.csv]         献立候補（料理一覧）
        ↓
献立作成 → 栄養価計算（レシピから集計）→ 発注指示書（食材を集計）
```
