# まとめ買い機能 設計メモ

## 1. 概要

日別発注とは別に、**まとめて発注する食材**（米・調味料・乾物・備蓄品など）の目安を計算する機能。

## 2. まとめ買いの分類

| 分類 | 例 | 発注の考え方 |
|------|-----|--------------|
| **米・主食** | 精白米、乾麺 | 月単位で大量発注。5kg袋など |
| **調味料** | 味噌、醤油、砂糖、油、だし | ボトル/パック単位。在庫切れで発注 |
| **乾物・海藻** | 乾燥わかめ、かつお節 | 袋単位。長期保存可能 |
| **備蓄品** | 缶詰、レトルト、非常食 | 定数在庫を維持。将来拡張 |

## 3. データモデル

### bulk_purchase_guide テーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| ingredient_id | INTEGER | 食材ID（PK, FK） |
| order_unit_g | REAL | 発注単位（g） |
| order_unit_name | TEXT | 表示名（例: 5kg袋、1本） |
| bulk_category | TEXT | 分類（米・主食/調味料/乾物・海藻/備蓄品） |

### マスタ例（bulk_purchase_guide.csv）

| 食材 | 発注単位 | 単位名 |
|------|----------|--------|
| 精白米 | 5000g | 5kg袋 |
| 米味噌 | 1000g | 1kg |
| 醤油 | 1000g | 1本（約1L） |
| 砂糖 | 1000g | 1kg |
| サラダ油 | 900g | 1本（900g） |
| かつお節・昆布だし | 50g | 1パック |
| 乾燥わかめ | 50g | 1袋 |

## 4. 計算ロジック

- 期間内の献立から使用量を集計（aggregateOrder と同様）
- bulk_purchase_guide に登録された食材のみ対象
- 発注目安 = ceil(総使用量g / 発注単位g)
