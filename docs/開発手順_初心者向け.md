## ステップ1：環境準備・プロジェクト構成

### 1.1 やること

- Pythonの動作確認
- プロジェクトフォルダの確認
- 仮想環境の作成（推奨）
- 必要なライブラリの準備

### 1.2 具体的な手順

#### ① Pythonのバージョン確認

ターミナルで以下を実行：

```bash
python3 --version
```

`Python 3.8` 以上が表示されればOK。

#### ② プロジェクトフォルダに移動

```bash
cd /path/to/kyuushoku-system
```

#### ②-2 phase1_python フォルダの作成（未作成の場合）

```bash
mkdir -p phase1_python
```

Phase1の実装は `phase1_python/` に配置する。

#### ③ 仮想環境を作成（推奨）

```bash
python3 -m venv venv
```

#### ④ 仮想環境を有効化

```bash
source venv/bin/activate   # Mac/Linux
# Windows の場合は: venv\Scripts\activate
```

プロンプトの先頭に `(venv)` が表示されればOK。

#### ⑤ 必要なライブラリをインストール

```bash
pip install openpyxl
```

- `openpyxl`：Excel（.xlsx）の読み書き用（ステップ5で使用）

#### ⑥ フォルダ構成の確認

以下の構成になっていることを確認（開発計画書v2.0のmonorepo構成）：

```
idea/  (project-root)
├── data/
│   ├── schema.json    # schema_version
│   ├── ingredients.csv
│   ├── dishes.csv
│   ├── recipe.csv
│   └── menus.csv
├── docs/
├── phase1_python/     # Phase1実装（ここに main.py など）
├── backend_go/        # Phase2以降（空でOK）
└── frontend_web/      # Phase3以降（空でOK）
```

#### ⑦ CSVローダー・参照整合性チェックの作成

`phase1_python/main.py` を作成する。以下を実装：

- `load_csv(filename)`：data/ 内のCSVを読み込み、リストの辞書として返す
- `check_reference_integrity(ingredients, dishes, recipe)`：recipe の dish_id / ingredient_id が ingredients / dishes に存在するか検証。違反時はエラー終了
- データパス：`PROJECT_ROOT = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))`、`DATA_DIR = os.path.join(PROJECT_ROOT, "data")`

`python phase1_python/main.py` で「食材: 19 件読み込みました」「参照整合性: OK」と表示されればOK。

### 1.3 完了の目安

- `python3 --version` でバージョンが表示される
- `pip list` で openpyxl が表示される
- 4つのCSVが読み込め、参照整合性チェックが通る

## ステップ3：献立登録（F1）

### 3.1 やること

- `python main.py create-menu --date 2026-04-01` で献立を登録する
- 各カテゴリから1品ずつ選び、menus.csv に保存する

### 3.2 具体的な手順

#### ① コマンドライン引数の解析

`argparse` を使って、`create-menu` と `--date` を受け取る。

#### ② 対話式で料理を選択

1. dishes.csv をカテゴリ別に分ける
2. 主食→主菜→副菜→汁物→デザートの順で、番号を選んでもらう
3. 5品揃ったら menus.csv に追記

#### ③ 同日登録の禁止

menus.csv を読み、指定日付が既に存在する場合はエラー終了。

### 3.3 コマンド例

```bash
python phase1_python/main.py create-menu --date 2026-04-01
```

### 3.4 完了の目安

- 5カテゴリすべて選択しないと登録できない
- 同日で2回登録しようとするとエラーになる
- menus.csv に正しく1行追加される

## ステップ4：栄養価計算（F2）

### 4.1 やること

- `python main.py calc --date 2026-04-01` で献立の栄養価を計算
- 料理栄養価 ＝ Σ（食材栄養価 × 使用量 ÷ 100）
- 丸め：エネルギー等は小数1位、食塩相当量は小数2位（四捨五入）

### 4.2 具体的な手順

#### ① 献立の取得

menus.csv から指定日付の行を取得。該当する dish_id を5つ取得。

#### ② 各料理の栄養価を計算

recipe.csv から該当 dish_id の食材と使用量を取得。  
ingredients.csv から栄養価を取得し、`栄養価 × 使用量 / 100` を合計。

#### ③ 献立全体の合計

5料理分の栄養価を足す。

#### ④ 目標値との比較表示

エネルギー650kcal、たんぱく質25g などと比較して表示。

### 4.3 コマンド例

```bash
python phase1_python/main.py calc --date 2026-04-01
```

### 4.4 出力例

```
エネルギー：612.4 kcal（目標650kcal：-37.6）
たんぱく質：24.3 g（目標25g：-0.7）
脂質：19.8 g
炭水化物：87.1 g
食塩相当量：2.8 g
```

## ステップ6：発注集計（F4）

### 6.1 やること

- `python phase1_python/main.py order --start 2026-04-01 --end 2026-04-30 --people 120`
- 期間内の献立から、レシピを横断して食材を集計
- 同一食材は合算、人数倍率を適用

### 6.2 具体的な手順

#### ① 期間内の献立を取得

menus.csv から start〜end の日付の行を取得。

#### ② 各献立の料理IDを取得

staple_id, main_id, side_id, soup_id, dessert_id の5つ。

#### ③ 各料理のレシピを集計

recipe.csv から該当 dish_id の ingredient_id と amount を取得。  
同一 ingredient_id は合算（amount を足す）。

#### ④ 人数倍率を適用

総使用量 ＝ 1人分合計 × 人数

#### ⑤ 出力

| 食材名 | 総使用量(g) | の形式で表示またはCSV出力
