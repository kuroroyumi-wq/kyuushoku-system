## 1. 目的

管理栄養士業務（献立作成・栄養計算・発注集計・帳票出力）を一元化し、以下を実現する：

- **転記作業の削減**
- **計算精度向上**
- **業務効率化**

※対象施設：保育園（3〜5歳児）

## 4. 献立の構成

献立は以下の5つのカテゴリで構成する。

| カテゴリ | 説明 | 例 |
|----------|------|-----|
| 主食 | エネルギー源となる炭水化物主体の料理 | ご飯、食パン、うどん |
| 主菜 | たんぱく質主体の料理 | 豚の生姜焼き、焼き鮭、卵焼き |
| 副菜 | 野菜・海藻などビタミン・ミネラル補給 | ほうれん草のお浸し、きんぴらごぼう |
| 汁物 | 水分・栄養補給 | 味噌汁、わかめスープ |
| デザート | 果物・乳製品など | りんご、ヨーグルト |

## 5. 計算仕様（確定ルール）

### 5.1 栄養計算式

```
料理栄養価 ＝ Σ（食材栄養価 × 使用量 ÷ 100）
献立栄養価 ＝ 料理栄養価の合計
```

### 5.2 丸めルール

| 項目 | 桁 | 方法 |
|------|-----|------|
| エネルギー | 小数1位 | 四捨五入 |
| たんぱく質 | 小数1位 | 四捨五入 |
| 脂質 | 小数1位 | 四捨五入 |
| 炭水化物 | 小数1位 | 四捨五入 |
| 食塩相当量 | 小数2位 | 四捨五入 |

※四捨五入で統一。

### 5.3 単位ルール

| 項目 | ルール |
|------|--------|
| 食材単位 | gで統一。mlは使用しない（液体もg換算値で登録） |
| レシピ | 1人分基準 |

### 5.4 人数倍率

発注集計時のみ人数を掛ける。

```
総使用量 ＝ 1人分使用量 × 人数
```

### 5.5 栄養価目標値（保育園3〜5歳児）

| 項目 | 目標値 |
|------|--------|
| エネルギー | 650 kcal/食 |
| たんぱく質 | 25 g/食 |
| 脂質 | 20 g/食（参考） |
| 炭水化物 | 85 g/食（参考） |
| 食塩相当量 | 3 g/食以下 |

## 6. 機能要件（詳細確定版）

### F1：献立登録

| 項目 | 内容 |
|------|------|
| コマンド | `python phase1_python/main.py create-menu --date 2026-04-01` |
| 入力 | 日付を指定。各カテゴリ1品を選択（対話式または引数指定） |
| 処理 | 各カテゴリ1品必須。同一日付登録は禁止（上書き不可）。重複時はエラー終了 |
| 出力 | menus.csvへ保存 |
| データ参照 | dishes.csv（献立候補） |

### F2：栄養価計算

| 項目 | 内容 |
|------|------|
| コマンド | `python phase1_python/main.py calc --date 2026-04-01` |
| 入力 | 日付を指定 |
| 処理 | 献立のレシピから栄養価を計算。目標値と比較 |
| 出力例 | エネルギー：612.4 kcal（目標650kcal：-37.6）<br>たんぱく質：24.3 g（目標25g：-0.7）<br>脂質：19.8 g<br>炭水化物：87.1 g<br>食塩相当量：2.8 g |
| データ参照 | ingredients.csv、recipe.csv、menus.csv |

### F3：献立表出力（Excel仕様確定）

| 項目 | 内容 |
|------|------|
| コマンド | `python phase1_python/main.py export --month 2026-04 --output 献立表.xlsx` |
| ファイル形式 | .xlsx |
| 用紙 | A4縦想定 |
| 必須列 | 日付、曜日、主食、主菜、副菜、汁物、デザート、エネルギー |
| シート構成 | 1か月1シート |

### F4：発注集計

| 項目 | 内容 |
|------|------|
| コマンド | `python phase1_python/main.py order --start 2026-04-01 --end 2026-04-30 --people 120` |
| 入力 | 開始日、終了日、人数 |
| 処理 | レシピを横断集計。同一ingredient_idは合算。人数倍率適用 |
| 出力フォーマット | \| 食材名 \| 総使用量(g) \| |
| データ参照 | ingredients.csv、recipe.csv、dishes.csv、menus.csv |

### F5：調理指示書（簡易版）

| 項目 | 内容 |
|------|------|
| スコープ | 献立に含まれる料理名・食材一覧・分量を出力。詳細調理手順はPhase2 |
| データソース | dishes.note、recipe.csv |

## 7. データ仕様（確定）

### 7.0 データ互換ポリシー（開発計画書v2.0準拠）

#### データの正（Source of Truth）

常に以下の論理モデルを正とする：ingredients、dishes、recipe、menus。

#### schema_version

すべてのデータ形式に schema_version を持たせる。

| 形式 | 格納場所 |
|------|----------|
| CSV | data/schema.json |
| SQLite | version テーブル |

#### 移行原則

- CSVは常にインポート可能であること
- 変換ツールは破壊的変更をしないこと
- 同一入力→同一出力が再現されること

### 7.1 食材マスタ（ingredients.csv）

| カラム | 型 | 必須 | 説明 |
|--------|-----|------|------|
| id | 整数 | ○ | 食材ID（一意） |
| name | 文字列 | ○ | 食材名 |
| ingredient_category | 文字列 | ○ | 食材分類 |
| energy | 数値 | ○ | エネルギー（100gあたり kcal） |
| protein | 数値 | ○ | たんぱく質（100gあたり g） |
| fat | 数値 | ○ | 脂質（100gあたり g） |
| carbohydrate | 数値 | ○ | 炭水化物（100gあたり g） |
| salt | 数値 | ○ | 食塩相当量（100gあたり g） |
| unit | 文字列 | ○ | 単位（gで統一） |
| waste_rate | 数値 | ○ | 廃棄率（0〜1、初期値0） |
| note | 文字列 | - | 備考 |

### 7.2 献立候補（dishes.csv）

| カラム | 型 | 必須 | 説明 |
|--------|-----|------|------|
| id | 整数 | ○ | 料理ID（一意） |
| name | 文字列 | ○ | 料理名 |
| menu_category | 文字列 | ○ | 献立カテゴリ（主食/主菜/副菜/汁物/デザート） |
| serving_size | 整数 | ○ | 1人分の総量（g） |
| note | 文字列 | - | 備考 |

### 7.3 レシピ（recipe.csv）

| カラム | 型 | 必須 | 説明 |
|--------|-----|------|------|
| dish_id | 整数 | ○ | 料理ID（dishes.id を参照） |
| ingredient_id | 整数 | ○ | 食材ID（ingredients.id を参照） |
| amount | 数値 | ○ | 1人分の使用量（g） |

**制約**：1料理最低1食材必須。amount > 0。

### 7.4 献立（menus.csv）

| カラム | 型 | 必須 | 説明 |
|--------|-----|------|------|
| date | 文字列 | ○ | 日付（YYYY-MM-DD形式） |
| staple_id | 整数 | ○ | 主食の料理ID |
| main_id | 整数 | ○ | 主菜の料理ID |
| side_id | 整数 | ○ | 副菜の料理ID |
| soup_id | 整数 | ○ | 汁物の料理ID |
| dessert_id | 整数 | ○ | デザートの料理ID |
| note | 文字列 | - | 備考 |

**制約**：date UNIQUE。NULL不可。

### 7.5 参照整合性

- recipe.dish_id は dishes.id に存在すること
- recipe.ingredient_id は ingredients.id に存在すること
- menus の staple_id / main_id / side_id / soup_id / dessert_id は dishes.id に存在すること

## 8. エラー処理（確定動作）

| 状況 | 動作 |
|------|------|
| CSV不存在 | 即時終了 |
| 参照整合性違反 | 処理停止 |
| レシピ未登録 | エラー終了 |
| 栄養価欠損 | 0扱い＋警告表示 |
| 同日献立登録 | エラー終了（上書き不可） |
| 5カテゴリ未選択 | 登録不可 |
